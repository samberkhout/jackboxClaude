GAME FLOW:
================================================================================
Phase 1: LOBBY
  └─ Host selects game & clicks Start

Phase 2: INPUT (Writing)
  └─ Each player gets 2 prompts
  └─ Prompts paired (same prompt → 2 different players)
  └─ Players write answers
  └─ Host advances manually

Phase 3: VOTE (Battle)
  ├─ Matchups created from answers
  ├─ Players vote on matchups (can't vote own answers)
  ├─ Auto-advances when all eligible voters vote
  └─ Auto-advances to REVEAL when all matchups done

Phase 4: REVEAL (Results)
  └─ Shows vote counts & percentages
  └─ Calculates scores (unanimous=500pts, regular=votes×100)

Phase 5: LEADERBOARD
  └─ Cumulative scores displayed
  └─ Host can start new round or reset

SCORING LOGIC:
================================================================================
Per matchup:
  - If option A has more votes than B: option_A.player gets points
  - If all votes are unanimous (all A or all B): 500 points
  - Otherwise: vote_count × 100 points

Example:
  - 4 players vote, 3 vote A, 1 vote B
  - Option A winner gets: 3 × 100 = 300 points
  - If all 4 voted A: 500 points (unanimous)

DATA FLOW:
================================================================================
INPUT PHASE:
  Player → submitInput → Server stores in submissions.INPUT
           ↓
           broadcast roomState
           ↓
           Host clicks "Next Phase"

VOTE PHASE:
  Player → submitVote({targetId: matchup.id, choice: 'A'|'B'})
           ↓
           Server stores in matchupVotes[matchupIndex]
           ↓
           Server checks: isMatchupVotingComplete()
           ├─ If YES: advanceMatchup() (auto-continue or REVEAL)
           └─ If NO: wait for more votes
           ↓
           broadcast roomState

REVEAL PHASE:
  Server → calculate scores → broadcast roomState with results
           ↓
           RevealPhase component displays results
           ↓
           Host clicks "Continue"
